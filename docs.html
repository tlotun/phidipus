01Phidipus là gì?

Phidipus là AI Agent tự trị chạy local trên macOS (M1–M4). Thay vì chỉ trả lời câu hỏi, Phidipus thực sự làm việc — nhìn màn hình, click chuột, gõ phím, tạo file, gửi tin nhắn Telegram — tất cả từ một câu lệnh ngôn ngữ tự nhiên của bạn.

Điểm khác biệt cốt lõi Phidipus KHÔNG phải chatbot. Nó là một agent thực thi — nhận lệnh qua Telegram, phân tích ý định, chọn workflow phù hợp và thực hiện các thao tác vật lý trên máy tính của bạn, sau đó báo cáo kết quả.

02Tầm nhìn dự án

Phidipus được xây dựng với một mục tiêu duy nhất: AI không chỉ nói, AI làm việc. Trong thế giới tràn ngập chatbot, Phidipus chọn con đường khác — trở thành một người thực thi thực sự trên máy tính của bạn.

Tên gọi — Phidipus Lấy cảm hứng từ loài nhện Phidippus — loài nhện nhảy nổi tiếng với thị lực sắc bén và phản xạ cực nhanh. Phidipus Agent cũng vậy: nhìn màn hình với độ chính xác cao, hành động tức thì từ một câu lệnh.

Vấn đề cần giải quyết

Các công cụ automation truyền thống (Zapier, n8n, Make) hoạt động tốt cho API-to-API workflows nhưng không thể tương tác với UI thực tế — chúng cần API endpoint. Trong khi đó, hầu hết ứng dụng doanh nghiệp Việt Nam (ERP nội bộ, phần mềm kế toán, giao diện web cũ) không có API.

Phidipus lấp đầy khoảng trống này: nó tương tác với UI như người dùng thực, nhưng được điều khiển bằng ngôn ngữ tự nhiên và học hỏi từ kinh nghiệm.

L1 — Orchestrator (Python / asyncio)

L1 là não bộ của Phidipus. Nó chạy trong Python process riêng, dùng asyncio cho concurrent task handling. L1 hoàn toàn không có quyền system-level — nó chỉ communicate với L2 qua Unix socket.

AgentLoop
Core Event Loop
Nhận lệnh từ Telegram/Admin, dispatch qua Priority Chain (P1→P5), quản lý concurrent workflows và timeout handling.
WorkflowExecutor
Node Runner
Chạy từng node theo thứ tự, xử lý variable injection {{var}}, error recovery, checkpoint save/restore.
MemoryManager
9-Layer Memory
Quản lý 9 hệ thống memory: ClickMemory, WorkflowMemory, SemanticMemory, EpisodicMemory, UserPreferences và nhiều hơn.
VisionRouter
Vision Dispatch
5-strategy cascade từ cache → DINO → Florence → VLM. Tự động chọn strategy tối ưu theo context và confidence.

L2 — Daemon (Swift / Objective-C)

L2 là một daemon macOS native viết bằng Swift, chạy với quyền Accessibility. Nó nhận JSON action requests từ L1 qua Unix socket và thực thi trên hệ thống thực. Tách biệt hoàn toàn khỏi Python để tránh GIL và đảm bảo timing chính xác.

Tại sao tách L1/L2? Nếu L1 crash (Python exception, OOM...), L2 daemon vẫn chạy và có thể cancel action đang thực hiện. Checkpoint system đảm bảo L1 có thể resume từ đúng node sau khi restart. Không bao giờ mất giữa chừng một workflow quan trọng.

P1 — SmartRouter (Exact Match)

SmartRouter là tầng nhanh nhất. Nó duy trì một dictionary ánh xạ trigger phrases → workflow_name. Khi nhận lệnh, nó normalize text (lowercase, strip punctuation, normalize Unicode Vietnamese) rồi lookup O(1).

Hiệu suất SmartRouter 45ms average response time. Tất cả lệnh quen thuộc như "báo cáo doanh số", "check email", "KPI dashboard" đều được handle tại đây mà không cần gọi bất kỳ AI model nào.

P2 — SmartAction (Regex + Rules)

Khi P1 miss, SmartAction dùng một tập 150+ regex patterns và rule-based logic để xử lý các lệnh có structure rõ ràng nhưng variable content. Ví dụ: "gửi tin nhắn cho [name]", "mở [app]", "tạo file [tên]".

P3 — Brain Think (Fine-tuned LLM)

Brain được gọi khi lệnh phức tạp, không có pattern rõ ràng, hoặc cần compose nhiều tasks. Brain phân tích intent và tạo workflow JSON. Với Brain Phase 1, output phải qua /approve flow trước khi thực thi.

python — brain approval flow
# Brain tạo workflow JSON brain_result = await brain.think(user_command) # Gửi preview cho user qua Telegram await telegram.send( f"🧠 Brain tạo workflow {len(nodes)} nodes:\n" f"{preview_text}\n\n" "/approve — Chấp nhận và chạy\n" "/reject — Từ chối" ) # Chờ user confirm (timeout 5 phút) response = await wait_for_approval(timeout=300) if response == "/approve": await executor.run(brain_result.workflow) # Lưu vào WorkflowLibrary cho lần sau workflow_lib.save(brain_result)

P4 — SkillForge (AI Code Generation)

Khi Brain tạo workflow nhưng cần custom logic không có sẵn trong 26 nodes, SkillForge được kích hoạt. Nó dùng Gemini Flash để generate Python code, sandbox nó trong Docker, test rồi wrap thành custom node.

P5 — Fallback

Khi tất cả tầng trên fail, Phidipus gửi Telegram message mô tả lý do không thể thực hiện và đề xuất cách user có thể làm thủ công hoặc điều chỉnh lệnh. Không bao giờ silent fail.

07// IPC Layer — Architecture

07IPC Communication Layer

L1 và L2 communicate qua Unix Domain Socket tại /tmp/phidipus.sock. Đây là cơ chế IPC nhanh nhất trên Unix, không qua network stack, latency <1ms.

Tại sao Unix Socket?

speed
Kernel-level Speed
Bypasses network stack hoàn toàn. Data transfer trong kernel memory, không copy qua userspace nhiều lần.
security
File System Permissions
Socket file có UNIX permissions. Chỉ processes với đúng UID mới connect được. Không accessible qua network.
reliability
Automatic Cleanup
OS tự cleanup socket khi process crash. L1 có thể detect L2 disconnect ngay lập tức qua SIGPIPE.

Message Protocol

python — ipc protocol
# L1 → L2: Action Request { "action": "mouse_click", "rx": 0.819, # relative x (0.0–1.0) "ry": 0.722, # relative y (0.0–1.0) "window_title": "Facebook", "button": "left", "verify": true, "timeout_ms": 500 } # L2 → L1: Action Result { "success": true, "actual_rx": 0.819, "actual_ry": 0.723, "elapsed_ms": 12, "accessibility_verified": true } # Supported actions từ L2: # mouse_click, mouse_double_click, mouse_right_click # keyboard_type, keyboard_hotkey, keyboard_press # screenshot (full / region / window) # app_launch, app_focus, app_quit # browser_execute_js, browser_get_dom # file_open, file_save_dialog

Relative Coordinates

Thay vì dùng pixel tuyệt đối (fragile khi resize), Phidipus dùng relative coordinates (0.0–1.0) relative to window size. Điều này giúp workflow chạy đúng ngay cả khi window size thay đổi hoặc display scale khác.

rx=0.50 ry=0.50
python — coordinate system
# Absolute → Relative conversion (L1 side) rx = pixel_x / window_width # 0.0 = left, 1.0 = right ry = pixel_y / window_height # 0.0 = top, 1.0 = bottom # Relative → Absolute (L2 side, handles retina) actual_x = int(rx * window.width * display_scale) actual_y = int(ry * window.height * display_scale) # Memory stores relative coords # → works across window resizes and retina switching

Pipeline Skill Forge (5 bước)

1

Task Analysis

Brain phân tích task và xác nhận không có node type nào phù hợp. Mô tả yêu cầu cho Skill Forge: input types, expected output, constraints, platform requirements.

2

AI Code Generation

Gemini Flash (hoặc qwen3:8b nếu offline) generate Python code cho custom skill. Prompt engineering đặc biệt đảm bảo code tuân theo Phidipus Skill API: async function, proper error handling, typed return.

3

Docker Sandbox Test

Code được chạy trong Docker container isolated: không có network access, không có filesystem ngoài /tmp, resource limits (RAM 256MB, CPU 1 core, timeout 10s). Nếu crash hoặc exception → gen lại.

4

User Approval

Skill được preview qua Telegram với code snippet và expected behavior. User /approve hoặc /reject. Sau approve, skill được đăng ký vào SkillLibrary với unique skill_id.

5

Runtime Integration

Skill được load như một dynamic module. WorkflowExecutor có thể dùng skill_id như node type bình thường. Skill được cache và tái sử dụng cho các workflow tương lai.

Skill API Contract

python — skill template
from phidipus.skill import SkillBase, SkillResult class CustomSkill(SkillBase): """Mô tả skill — Brain dùng để match""" skill_name = "my_custom_skill" skill_version = "1.0.0" async def execute( self, config: dict, context: dict # biến từ nodes trước ) -> SkillResult: # Skill logic ở đây result = await do_something(config["input"]) return SkillResult( success=True, output=result, next_context={"skill_output": result} )

Security Model

  • Sandboxed execution — Mọi code chạy trong Docker container isolated, không có quyền access host filesystem hay network
  • Static analysis — Trước khi sandbox, code được scan bằng AST parser để detect suspicious patterns (os.system, subprocess, socket...)
  • Resource limits — RAM 256MB, CPU 1 core, timeout 10 giây — không thể abuse host resources
  • User approval bắt buộc — Không có skill nào được install mà không có explicit /approve từ user

Technical Requirements cho Phase 4

num_ctx
Context Window Upgrade
Từ 8,192 → 16,384 tokens. Cần để fit 5 turns chat history + current task + workflow context. RAM tăng nhẹ (+1GB cho Brain).
training data
200+ Multi-turn Samples
Cần ~200 samples đặc biệt: multi-turn conversations với pronoun resolution, task reference, "sửa lại", "làm thêm", "thêm bước X".
chat buffer
5-Turn History Buffer
Agent loop inject 5 turns gần nhất vào Brain context. Kết hợp với SessionState memory để track task outputs trong cùng hội thoại.
coreference
Coreference Resolution
Brain học map "cái đó", "nó", "task trước", "kết quả vừa rồi" → đúng entity trong history. Requires dedicated training examples.
